# vitrado_v2 — Work Summary

Full-stack affiliate/partner management platform built on **PHP 8 / Phalcon 5** (backend) and **Angular** (frontend, three separate projects: `v2`, `superuser`, `advertiser`). The platform manages affiliate networks, advertiser partnerships, KWK magic links, statistics, invoicing, user authentication via Azure AD, and a MICOS integration — deployed via Docker containers on Kubernetes.

---

## Backend

### Azure AD SSO Integration

Implemented the Azure Active Directory single sign-on flow from scratch. Built the OAuth2 authorization code flow including role extraction from Azure App Role Assignments, token refresh, and MICOS check integration. Worked through extensive debugging to get role/scope claims reading correctly from the Azure Graph API. Added database migrations for user changes required by the new auth model. Created and configured SSL directories and config values for certificate-based authentication.

Updated the integration when the `vit_azure` library was introduced as the canonical shared implementation, migrating v2 to use its standardised classes and configuration.

### Session Management & Appserver Decoupling

Decoupled v2's session from the appserver by sharing session state through Redis instead of a directly coupled PHP session. Implemented Redis-backed session handling (`RedisAdapter`) so v2 and appserver can operate independently while still serving authenticated users consistently. Added middleware-level session start and handling so controllers and services no longer manage sessions themselves.

Implemented appserver cookie auto-expiry: when the appserver session expires, v2 detects this, forces a new session establishment (ping), and refreshes the session ID automatically without requiring user re-login. Added XSRF token reset on logout and ensured the XSRF cookie is cleared on session destroy.

### API Middleware & Security Refactoring

Built `ApiMiddleware.php` as the single point of responsibility for all API security: ACL enforcement, XSRF validation, referrer checks (with bypass for internal API token requests), request logging, and session handling. Removed all security-related logic from `ControllerBase` and `CallController` — moving it entirely into middleware to centralise security and reduce controller complexity.

Refactored base service to obtain the DI container through a method rather than direct access, removed auto session start from services, and added HTTP status code constants.

### API Token Authentication (REST API)

Designed and implemented a token-based REST API authentication system for external integrations:
- Created `ApiConstants`, token models, and a `TokenGenerator` utility.
- Added a `RestApiAuthenticator` middleware to validate bearer tokens.
- Built controller endpoints for token generation, deletion, and user-level token management.
- Implemented a superuser-only API token type with separate ACL registration.
- Added a database migration for the token table.
- Integrated try-catch and XSRF bypass for token deletion endpoints.

### Logging System Refactoring

Replaced ad-hoc logging across the codebase with a unified logger using a `Loggable` trait. Updated all services and jobs to use the logger, with log names derived from the calling class. Separated job logs into their own log files per job type. Allowed custom log names without class existence checks, enabling flexible log routing.

### Magiclink Service (KWK)

Built the full magiclink feature for KWK affiliate partners:
- `PartnerTrackingLinksService` and `KwkService` backend logic for generating, validating, updating, and sanitising tracking links.
- `MagicLinkModel` and interface definitions.
- ACL rights registration and paginator-based listing.
- Validation and input sanitisation on update.
- Small delay on update to ensure consistency.

### Advertiser & Partner Management

- Built `AdvertiserCreateService` and `AdvertiserProfileValidator` for advertiser onboarding and profile management.
- Extended `PartnershipsService` to support `GROUP BY` queries with correct partnership status filtering.
- Implemented sorting improvements and user-type filtering for the users list (ignoring KWK partners in standard partner views).
- Added a `VITDEV-1302` special character normalisation method to replace accented characters with ASCII equivalents in personal data handling, applied across all methods handling user personal data.

### Ad Guids & Programs Service

Updated `AdGuidsService` and `AdsService` to mark ad GUIDs and ads as disabled/deleted rather than removing records from the database. Updated the link between partnership updates and ad GUID state synchronisation. Updated `ProgramsService` to remove HMVC dependency and simplify.

### Awin Integration

Added `AwinService` and `AwinCacheJob` to integrate with the Awin affiliate network API. Implemented an initial request to obtain the session cookie before subsequent API calls, with filtering and proper response interpretation.

### Reports & CSV Invoicing

Built `ReportsService` with report export functionality. Replaced iframe-based file download with a proper backend-served download flow. Fixed multi-file download handling and transaction download file naming.

### Database Migrations

Created multiple Phalcon database migrations to support new features: Azure user model changes, API token table, KWK magic link table, character normalisation support.

### Cron Job Management

Built and maintained several background cron jobs: `StatCompactClicks`, `StatQueueProcess`, `StatQueueProcessRating`, `StatRebuildByDate`, `ProcessMicosQueue`, `InvoicesCsvDownload`, `InvoicesCsvProcess`, `SalesCallbackJob`, `DeletePartners`, `FixKwkPartnerships`. Added job locking to prevent duplicate concurrent execution. Fixed a bug where invalid parameters caused jobs to lock permanently.

Added cron scheduling (`screen`-less approach after `screen` stop was removed) and cleaned up the cron file.

### MICOS Integration

Extended `MicosApiConsumer` and `MicosApiService` to handle additional error states and edge cases in the MICOS queue processing pipeline. Added explicit MICOS response code checks and error treatment.

### Transactions & Email Filters

Fixed the ads query and the transactions email filter. Fixed the transactions email filter for the double-filtering case.

### Superuser Panel Redirect

Implemented last-panel redirect for the superuser: stores the previous Angular panel route in session and restores it when the superuser navigates back, preventing redirect loops.

## DevOps

### Docker Multi-Container Setup

Maintained a Docker setup with separate images: a PHP base image (`Dockerfile.base`), a PHP application image (`Dockerfile`), and an Nginx image (`Dockerfile.nginx`). Managed per-environment `php.ini`, `opcache.ini`, and PHP-FPM pool configurations. Maintained `docker-compose.yaml`, `docker-compose.stage.yaml`, and `docker-compose.production.yaml`.

Added an Nginx logs folder entry to the compose setup. Fixed PHP path references in the Nginx FastCGI config. Removed double-copy steps in Dockerfile that caused incorrect file ownership.

### Session Storage via Redis

Configured Redis as the PHP session backend for the v2 application container, replacing filesystem sessions, to support the decoupled session sharing architecture with appserver.

### SSH2 PHP Extension

Added the `ssh2` PHP extension and required packages to the Docker base image to support SFTP-based operations.

### Nginx Configuration

Updated Nginx log names per environment (stage-specific log file naming). Updated Nginx FastCGI configs and application path references.

## CI/CD

### Container Builder Workflow

Built `container_builder.yml` using the shared `vit-workflow` `platform_builder` reusable workflow. Builds base PHP, dev PHP, and Nginx images on feature branch pushes with change detection (`filter_changes.yml`). Supports `force-build` via commit message keyword and manual dispatch.

### Deploy Workflow

Built `deploy.yml` triggered on pushes to `stage` and `production`. Orchestrates:
1. Angular build (all three projects) via `build_angular.yml`.
2. PHP container build (parallel with Angular).
3. Nginx container build (depends on Angular artefact, extracts `superuser`, `app`, `advertiser` targets from the archive).
4. Deployment over SSH:
   - Activates **maintenance mode** via CLI before stopping containers.
   - Stops current containers, pulls new images, starts updated containers.
   - Runs database migrations inside the new PHP container.
   - Deactivates maintenance mode when complete.
   - Runs ACL reset inside the container post-deploy.
   - Combined stage and production deployment in a single parameterised deploy job.

### Platform Builder (Reusable Workflow)

Maintained the v2-specific `platform_builder.yml` reusable workflow used by the container builder and deploy workflows, handling multi-arch builds via the `create_containers` action from `vit-workflow`.

### Angular Build Workflow

Maintained `build_angular.yml` for building and archiving all three Angular projects (`superuser`, `app`, `advertiser`) per environment, uploading the artefact for use by the Nginx container build step.

### Periodic Check

Configured `periodic_check.yml` for scheduled daily vulnerability scanning of all container images.

### ESLint & Code Quality

Added ESLint configuration to the Angular projects for type-hinting and code quality enforcement. Ran linting across all files as part of the Angular upgrade.
